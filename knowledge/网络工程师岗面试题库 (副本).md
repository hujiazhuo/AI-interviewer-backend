# 网络工程师 面试题库

------

# 一、网络基础（30 题）

### 1. OSI 七层与 TCP/IP 五层模型及各层作用

**答**：

OSI 七层：物理层、数据链路层、网络层、传输层、会话层、表示层、应用层。

TCP/IP 五层：物理层、数据链路层、网络层、传输层、应用层。

- 物理层：传输比特流，定义电气 / 物理规范。
- 数据链路层：成帧、MAC 寻址、差错校验、交换机工作。
- 网络层：IP 寻址、路由选择、路由器工作。
- 传输层：端到端通信、TCP/UDP、端口号。
- 应用层：为应用程序提供网络服务（HTTP、FTP、DNS 等）。

### 2. TCP 和 UDP 的区别及应用场景

**答**：

- TCP：面向连接、可靠传输、流量 / 拥塞控制、开销大；用于 HTTP、HTTPS、FTP、邮件。
- UDP：无连接、不可靠、速度快、开销小；用于视频直播、DNS、游戏、语音。

### 3. TCP 三次握手详细过程及原因

**答**：

1. 客户端发送 SYN 到服务端，请求连接。

2. 服务端回复 SYN+ACK，确认并同意连接。

3. 客户端回复 ACK，连接建立。

   

   目的：确保双方收发能力正常，避免无效连接占用服务端资源。

### 4. TCP 四次挥手详细过程及原因

**答**：

1. 主动方发送 FIN，请求关闭。

2. 被动方回复 ACK，确认收到。

3. 被动方发送 FIN，准备关闭。

4. 主动方回复 ACK，连接关闭。

   

   原因：双方需独立关闭发送通道，数据传输可能不对称。

### 5. SYN 洪水攻击原理及防御

**答**：

攻击者发送大量 SYN 不回复 ACK，占满半连接队列。

防御：SYN-Cookie、限制连接速率、缩短超时、防火墙清洗。

### 6. TCP 可靠传输机制

**答**：序列号、确认应答、超时重传、滑动窗口、流量控制、拥塞控制。

### 7. 滑动窗口、流量控制、拥塞控制

**答**：

- 滑动窗口：控制发送量，提高传输效率。
- 流量控制：接收方告知发送方发送速度，防止缓冲区溢出。
- 拥塞控制：根据网络拥堵情况调整发送速率。

### 8. ARP 协议原理及 ARP 欺骗

**答**：

ARP 通过 IP 地址获取 MAC 地址。

ARP 欺骗：伪造 ARP 报文篡改网关 / 主机 MAC，实现监听、断网。

防御：DAI、静态 ARP、端口安全。

### 9. 私有 IP 地址范围

**答**：

- 10.0.0.0 ~ 10.255.255.255（10/8）
- 172.16.0.0 ~ 172.31.255.255（172.16/12）
- 192.168.0.0 ~ 192.168.255.255（192.168/16）

### 10. 子网划分、VLSM、CIDR

**答**：

- 子网划分：将大网段拆分为小网段，缩小广播域。
- VLSM：变长子网掩码，不同网段可使用不同掩码。
- CIDR：无类别域间路由，使用 `/前缀` 表示网段。

### 11. 网关作用

**答**：跨网段通信的出口，没有网关无法访问其他网段。

### 12. DNS 工作流程、递归与迭代区别

**答**：

流程：客户端 → 本地 DNS → 根 DNS → 顶级域 DNS → 权威 DNS → 返回结果。

- 递归：本地 DNS 替客户端完成全部查询。
- 迭代：客户端逐级查询各个 DNS 服务器。

### 13. DHCP 四步报文

**答**：Discover → Offer → Request → ACK。

### 14. NAT 作用及类型

**答**：

作用：节省公网 IP、隐藏内网拓扑。

类型：静态 NAT、动态 NAT、PAT（端口多路复用）。

### 15. ICMP 协议作用

**答**：网络探测、错误反馈，ping、tracert 基于 ICMP。

### 16. MTU 与 MSS

**答**：

- MTU：数据链路层最大传输单元，默认 1500。
- MSS：TCP 最大分段大小，MSS = MTU - 40。

### 17. 广播域与冲突域

**答**：

- 冲突域：同一域内同时发送数据会产生冲突；交换机每个端口一个冲突域。
- 广播域：接收同一广播报文的范围；路由器隔离广播域。

### 18. 常用网络排错命令

**答**：ping、tracert、arp、ipconfig、netstat、telnet、ssh、route。

### 19. HTTP 与 HTTPS 区别

**答**：

- HTTP：80 端口，明文传输。
- HTTPS：443 端口，加密传输、身份认证、防篡改。

### 20. HTTPS 加密原理

**答**：非对称加密协商会话密钥，对称加密传输业务数据，证书验证身份。

### 21. 端口号范围划分

**答**：

- 0~1023：知名端口
- 1024~49151：注册端口
- 49152~65535：动态 / 私有端口

### 22. 长连接与短连接

**答**：

- 短连接：一次请求 / 响应后断开。
- 长连接：一次连接可多次请求，HTTP 1.1 默认开启。

### 23. 网络丢包排查思路

**答**：检查物理链路、端口状态、双工不匹配、环路、拥塞、防火墙策略、光模块、路由震荡。

### 24. 网络延迟高排查

**答**：检查路由路径、带宽拥塞、设备 CPU、NAT 开销、传输距离、队列调度。

### 25. RARP、免费 ARP、代理 ARP

**答**：

- RARP：MAC → IP。
- 免费 ARP：开机广播自己 IP，检测冲突。
- 代理 ARP：网关代替主机应答 ARP 请求。

### 26. 公有 IP 与私有 IP 区别

**答**：公有 IP 全球唯一可访问互联网；私有 IP 仅内网使用，需 NAT 访问外网。

### 27. 默认路由与浮动路由

**答**：

- 默认路由：0.0.0.0/0，缺省路由。
- 浮动路由：备份路由，优先级更低，主路由故障时启用。

### 28. 路由优先级与度量值

**答**：

- 优先级：不同路由协议之间优选，值越小越优先。
- 度量值：同一协议内选最优路径，值越小越优先。

### 29. 常见路由优先级（华为）

**答**：直连 0 → 静态 60 → OSPF 110 → RIP 120。

### 30. 跨网段通信完整流程

**答**：主机查路由 → ARP 请求网关 → 网关查路由 → 转发到目标网段 → 回包原路返回。

# 二、交换技术（35 题）

### 1. 交换机工作原理

**答**：学习源 MAC 地址 → 查询 MAC 地址表 → 转发 / 泛洪。

### 2. 交换机、集线器、路由器区别

**答**：

- 集线器：物理层，共享介质，半双工。
- 交换机：数据链路层，隔离冲突域，全双工。
- 路由器：网络层，隔离广播域，跨网段转发。

### 3. VLAN 作用

**答**：隔离广播域、提升安全性、简化组网、灵活划分业务。

### 4. Access/Trunk/Hybrid 端口区别

**答**：

- Access：接入终端，仅允许一个 VLAN，发送 Untag 帧。
- Trunk：中继链路，允许多个 VLAN，发送 Tag 帧。
- Hybrid：华为特有，可自定义 Tag/Untag。

### 5. 802.1Q 协议作用

**答**：为以太网帧添加 VLAN 标签，实现跨交换机 VLAN 透传。

### 6. 三层交换机与路由器区别

**答**：三层交换机高速硬件转发，用于内网；路由器支持 NAT、VPN、广域网接口。

### 7. STP 协议作用

**答**：消除二层环路，修剪冗余链路形成无环树。

### 8. STP 选举：根桥、根端口、指定端口

**答**：

- 根桥：桥 ID 最小（优先级 + MAC）。
- 根端口：非根桥到达根桥最优路径端口。
- 指定端口：每网段唯一负责转发的端口。

### 9. STP 端口状态

**答**：Disabled、Blocking、Listening、Learning、Forwarding。

### 10. RSTP 对 STP 的改进

**答**：精简端口状态、快速收敛、减少拓扑变更延时。

### 11. MSTP 作用

**答**：多实例，不同 VLAN 映射不同生成树，实现负载分担。

### 12. 二层环路危害

**答**：广播风暴、MAC 漂移、设备 CPU 占用高、网络瘫痪。

### 13. 链路聚合 Eth-Trunk/LACP 作用

**答**：增加带宽、链路冗余、负载均衡。

### 14. 静态 LACP 与动态 LACP

**答**：静态不协商；动态通过 LACP 协议协商，更可靠。

### 15. MAC 地址漂移原因及排查

**答**：环路、双主、错接链路。排查：查看日志、shutdown 端口、检查拓扑。

### 16. 端口安全功能

**答**：限制 MAC 学习数量、违规动作保护端口。

### 17. 风暴抑制作用

**答**：抑制广播 / 组播 / 未知单播风暴，保护设备。

### 18. 端口镜像作用

**答**：流量复制，用于抓包分析、监控。

### 19. VLAN 间通信三种方式

**答**：单臂路由、三层交换机、路由器物理口分网段。

### 20. 堆叠、集群、级联区别

**答**：堆叠多台虚拟为一台；集群多主备份；级联简单扩展端口。

### 21. 同 VLAN 不通排查

**答**：端口 VLAN 配置、Trunk 透传、ACL、链路、MAC 表、ARP。

### 22. 不同 VLAN 不通排查

**答**：三层接口、SVI、路由、网关、ACL。

### 23. DHCP Snooping 作用

**答**：防止非法 DHCP 服务器，绑定 IP+MAC + 端口 + VLAN。

### 24. DAI 动态 ARP 检测

**答**：基于 DHCP Snooping 绑定表，防御 ARP 欺骗。

### 25. IPSG IP 源防护

**答**：防止 IP 伪造、IP 地址冲突。

### 26. 交换机环路排查步骤

**答**：查看 MAC 漂移日志 → shutdown 冗余口 → 检查 Trunk → 检查终端回路 → 开启 STP。

### 27. 802.1X 认证作用

**答**：端口级接入认证，提升内网安全。

### 28. 交换机初始化配置

**答**：主机名、密码、管理 IP、默认路由、SNMP、日志。

### 29. 交换机密码恢复

**答**：重启进入 BootROM → 清除配置 → 重启 → 重新配置。

### 30. 交换机 CPU 高原因

**答**：广播风暴、ARP 攻击、协议报文、环路、病毒。

### 31. 端口 UP/DOWN 闪断原因

**答**：光模块不匹配、链路松动、双工不匹配、对端震荡、供电不稳。

### 32. 交换机 QoS 作用

**答**：保障关键业务带宽、优先级调度、流量整形。

### 33. 堆叠分裂影响

**答**：多主冲突、路由震荡、业务中断。

### 34. 本地镜像与远程镜像

**答**：本地镜像同设备；远程镜像跨设备镜像。

### 35. 交换机日志查看要点

**答**：端口状态、MAC 漂移、ARP 攻击、CPU、环路、协议状态。

# 三、路由技术（45 题）

### 1. 路由器工作原理

**答**：查路由表 → 最长匹配 → 转发到下一跳 / 出接口。

### 2. 静态路由与动态路由区别

**答**：静态手动配置、无开销、适合小型网络；动态自动学习、适合大型网络。

### 3. 默认路由应用场景

**答**：出口路由、末梢网络、缺省转发。

### 4. 浮动路由作用

**答**：主路由故障时自动切换备份路由。

### 5. 路由表构成

**答**：目的网络、掩码、协议、优先级、度量、出接口、下一跳。

### 6. 路由优先级（华为）

**答**：直连 0 → 静态 60 → OSPF 110 → IS-IS 15 → RIP 120。

### 7. 路由度量值作用

**答**：同一协议内选择最优路径。

### 8. RIP 协议特点

**答**：距离矢量、跳数度量、最大 15 跳、周期更新、适合小型网络。

### 9. RIP 防环机制

**答**：水平分割、毒性逆转、触发更新、最大跳数。

### 10. OSPF 协议特点

**答**：链路状态、区域化、组播更新、快速收敛、适合大型网络。

### 11. OSPF 区域 0 作用

**答**：骨干区域，所有区域必须连接区域 0，负责区域间路由转发。

### 12. DR、BDR 选举作用

**答**：减少广播网邻接关系，降低 LSA 泛洪。

### 13. OSPF 五种报文

**答**：Hello、DBD、LSR、LSU、LSAck。

### 14. OSPF 邻接建立过程

**答**：Down → Init → 2-Way → ExStart → Exchange → Loading → Full。

### 15. OSPF 开销计算

**答**：接口开销 = 10^8 / 带宽。

### 16. OSPF 虚连接作用

**答**：修复非骨干区域未直连区域 0 的问题。

### 17. OSPF 与 RIP 区别

**答**：OSPF 链路状态、无环、快收敛、支持大规模；RIP 距离矢量、有环、慢收敛。

### 18. IS-IS 与 OSPF 区别

**答**：IS-IS 更稳定，运营商常用；OSPF 企业常用。

### 19. BGP 协议作用

**答**：自治系统间路由协议，控制路由策略。

### 20. IBGP 与 EBGP 区别

**答**：EBGP 跨 AS；IBGP 同 AS，需全互联或路由反射器。

### 21. BGP 常用属性

**答**：Origin、AS-Path、Next-Hop、Local_Pref、MED。

### 22. BGP 选路原则（前 8）

**答**：优先级 → Local_Pref → 本地始发 → AS-Path 短 → Origin → MED → EBGP＞IBGP → IGP 开销。

### 23. 路由引入 import-route

**答**：不同协议互相引入，需配置过滤避免环路。

### 24. 策略路由 PBR 作用

**答**：基于源 IP、应用、端口自定义转发路径。

### 25. 路由过滤工具

**答**：ACL、前缀列表、Route-Policy、过滤器。

### 26. 黑洞路由作用

**答**：丢弃特定流量，防攻击、引流。

### 27. 等价路由 ECMP

**答**：多条同优先级路由负载分担。

### 28. 单臂路由优缺点

**答**：优点：低成本实现 VLAN 间路由；缺点：瓶颈、单点故障。

### 29. VRRP 协议作用

**答**：网关冗余，多台路由器虚拟一个网关。

### 30. VRRP 主备选举

**答**：优先级高为主，相同则接口 IP 大为主。

### 31. BFD 作用

**答**：快速故障检测，毫秒级感知链路中断。

### 32. 标准 ACL 与扩展 ACL

**答**：标准仅过滤源 IP；扩展过滤源 / 目的 IP、端口、协议。

### 33. GRE 隧道作用

**答**：三层 VPN，封装跨网传输。

### 34. IPSec VPN 作用

**答**：加密传输，保障公网数据安全。

### 35. 路由不通排错步骤

**答**：查直连 → 查网关 → 查路由表 → 查下一跳 → 查 ACL → 查 NAT → 查链路。

### 36. 能 ping 网关但 ping 不通外网

**答**：缺省路由、NAT、ACL、路由黑洞、运营商链路。

### 37. OSPF 邻居起不来原因

**答**：区域 ID、认证、掩码、Hello/Dead、静默接口、物理 down。

### 38. 路由环路产生原因

**答**：双向引入、过滤不当、静态路由配置错误。

### 39. 路由器 NAT Server 作用

**答**：内网服务器映射公网 IP 供外网访问。

### 40. 路由震荡原因

**答**：接口不稳定、协议参数不合理、链路质量差。

### 41. 静态路由配置注意事项

**答**：下一跳可达、避免黑洞、配置备份、合理优先级。

### 42. 路由汇总作用

**答**：减小路由表规模、加快收敛、提升稳定性。

### 43. 组播路由基础

**答**：一对多发送，用于视频、直播。

### 44. MPLS 作用

**答**：多协议标签交换，提升转发效率，用于企业专线。

### 45. 出口双线路设计

**答**：主备 / 负载分担、NAT、策略路由、路由探测。

# 四、网络安全（25 题）

### 1. 防火墙三大区域

**答**：Trust、Untrust、DMZ。

### 2. 包过滤与状态检测区别

**答**：包过滤仅检查报文头；状态检测跟踪会话，更安全。

### 3. ACL 作用

**答**：访问控制、流量过滤、路由过滤、策略匹配。

### 4. 常见网络攻击

**答**：ARP 欺骗、DHCP 欺骗、SYN 洪水、DDoS、DNS 劫持、端口扫描、暴力破解。

### 5. IPS/IDS 区别

**答**：IDS 检测告警；IPS 在线阻断。

### 6. VPN 类型

**答**：IPSec、SSL、L2TP、GRE。

### 7. 对称加密与非对称加密

**答**：对称加密快；非对称加密安全，用于密钥协商。

### 8. CA 证书作用

**答**：身份认证、防篡改、防伪造。

### 9. 802.1X 认证流程

**答**：认证请求 → 身份上报 → 认证 → 授权 / 拒绝。

### 10. 零信任安全理念

**答**：默认不信任，持续验证、最小权限。

### 11. WAF 作用

**答**：Web 应用防火墙，防御 SQL 注入、XSS。

### 12. 堡垒机作用

**答**：统一运维、操作审计、权限管控。

### 13. 网闸作用

**答**：物理隔离，内外网安全数据摆渡。

### 14. 安全组与 ACL 区别

**答**：安全组有状态；ACL 无状态。

### 15. 端口扫描防御

**答**：防火墙禁用 ICMP、限制连接、入侵检测。

### 16. 弱口令安全策略

**答**：复杂度、定期更换、锁定机制、多因素认证。

### 17. 日志审计作用

**答**：追溯、排查、合规、取证。

### 18. 等保 2.0 基本要求

**答**：物理、网络、主机、应用、数据、管理。

### 19. 网络准入 NAC 作用

# 五、无线 WLAN（20 题）

### 1. 802.11a/b/g/n/ac/ax 区别

**答**：ax 即 WiFi 6，速度更快、并发更高、延迟更低。

### 2. 胖 AP 与瘦 AP 区别

**答**：胖 AP 独立工作；瘦 AP 由 AC 统一管理。

### 3. AC+Fit AP 架构

**答**：AC 集中管理，Fit AP 提供接入，CAPWAP 隧道。

### 4. SSID 作用

**答**：无线网络名称。

### 5. 无线漫游原理

**答**：终端在 AP 间无缝切换，保持 IP 不变。

### 6. 2.4G 与 5G 区别

**答**：2.4G 穿墙好、干扰大；5G 速度快、干扰小。

### 7. 无线加密方式

**答**：WEP 不安全；WPA2/WPA3 企业版安全。

### 8. CAPWAP 协议作用

**答**：AC 与 AP 间控制与数据隧道。

### 9. 无线负载均衡

**答**：均衡用户数、流量，提升体验。

### 10. 无线用户隔离

**答**：禁止终端互访，提升安全。

### 11. 无线覆盖优化

**答**：功率、信道、布放位置、天线角度。

### 12. 信道规划原则

**答**：2.4G 用 1、6、11；5G 自动信道。

### 13. 无线丢包原因

**答**：干扰、弱信号、过载、漫游切换、同频干扰。

### 14. 802.11k/v/r

**答**：k 扫描邻居；v 快速切换；r 快速漫游。

### 15. WIPS 作用

**答**：检测非法 AP、钓鱼 AP、入侵。

### 16. 无线 VLAN 规划

**答**：按业务 / 访客 / 办公划分。

### 17. 访客网络设计

**答**：隔离、限速、认证、时效。

### 18. 无线认证方式

**答**：PSK、802.1X、Portal、短信、微信。

### 19. 无线故障排查

**答**：信号、干扰、关联、认证、DHCP、路由。

### 20. 企业 WLAN 设计要点

**答**：冗余 AC、POE 供电、信道规划、漫游、安全。
